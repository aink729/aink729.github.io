Background

비영리 기부 단체의 온라인 기부 플랫폼을 신규 구축하는 프로젝트입니다. 다양한 연령대의 사용자를 대상으로 하며, 모바일 사용 비율이 70% 이상인 서비스 특성상 모바일 퍼스트 반응형 UI 설계가 핵심이었습니다. 유사한 기부 항목 페이지가 50개 이상 존재하여, 일관된 UI 컴포넌트 시스템 구축이 병행 과제로 요구되었습니다.

Responsibilities

  • 모바일 퍼스트 기반 전 페이지 반응형 HTML · CSS 마크업
  • 버튼, 인풋, 카드, 탭, 모달, 토스트 등 공통 컴포넌트 30개 이상 설계 및 마크업 가이드 작성
  • clamp()를 활용한 유동형 타이포그래피 시스템 구현
  • IE11 포함 크로스브라우저 대응 (Polyfill, Fallback CSS)
  • 기부 페이지 공통 레이아웃 템플릿화로 신규 페이지 제작 시간 단축

Key Challenge

기부 페이지마다 유사하지만 미묘하게 다른 레이아웃이 반복되어, 담당자마다 마크업 방식이 달라지고 유지보수와 QA 비용이 계속 늘어나는 상황이었습니다.

레이아웃 패턴을 분석해 4가지 베이스 템플릿으로 분류하고, BEM 방법론을 도입하여 클래스 네이밍과 컴포넌트 경계를 팀 전체에 표준화했습니다.

신규 기부 페이지 제작 시간 기존 대비 70% 단축. 마크업 일관성 확보로 QA 사이클 단축, 온보딩 시 신규 퍼블리셔 적응 시간도 감소.

Retrospective

컴포넌트 시스템의 가치를 직접 체감한 프로젝트였습니다. 초기에 기준을 잡는 데 시간이 걸렸지만, 이후 유사 페이지를 빠르게 찍어낼 수 있었고 QA팀도 일관된 패턴 덕분에 테스트 범위를 좁힐 수 있었습니다. 퍼블리셔도 단순 마크업을 넘어 시스템적 사고를 해야 팀 전체의 생산성에 기여할 수 있다는 것을 배웠습니다.
